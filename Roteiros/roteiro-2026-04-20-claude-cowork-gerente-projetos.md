## Bloco 1 — O Que É o Claude Cowork e Como Funciona (2min–5min)

**[Visual: demo do interface Cowork, destacando thread persistente]**

### O conceito central: thread persistente

O Cowork funciona com o que a Anthropic chama de **thread de agente persistente**.

Diferente de um chat normal — onde cada conversa começa do zero — o Cowork mantém contexto ao longo do tempo. Você define um objetivo, o agente começa a trabalhar, e você pode fechar o computador. Quando você volta, ele reporta o que fez.

Pensa assim: é como contratar um assistente que tem memória de longo prazo e que você não precisa ficar gerenciando em tempo real.

### O que dá para delegar

Com o Cowork, você pode delegar tarefas que têm múltiplos passos e levam tempo:

- **Pesquisa**: "Pesquise os 10 maiores players de micro SaaS no Brasil e me manda um relatório com dados de crescimento"
- **Monitoramento**: "Toda vez que a Anthropic lançar algo novo, me manda um resumo"
- **Análise de dados**: "Analisa esses 50 arquivos e me diz quais têm inconsistências"
- **Produção de conteúdo**: "Cria 5 opções de título para esse roteiro com base nos dados de CTR que eu te mandei"

### Enterprise vs. plano padrão

Para usuários individuais: thread persistente + execução assíncrona de tarefas.

Para Enterprise: analytics expandidos + controle de acesso por papel (quem pode ver e editar o quê). Útil para equipes que querem delegar tarefas para o Cowork sem dar acesso total ao sistema.

## Bloco 2 — O Ecossistema Fechado: Por Que Isso É Maior Do Que Parece (5min–9min)

**[Visual: diagrama do loop Design → Code → Cowork → Routines]**

### A jogada estratégica da Anthropic

Vou te mostrar algo que a maioria das análises está ignorando.

Cada produto da Anthropic foi projetado para se conectar com os outros:

1. Você usa o **Claude Design** para criar um protótipo de um produto
2. O Design gera um **handoff bundle** que vai direto para o **Claude Code**
3. O Claude Code escreve o código, e você configura **Routines** para automatizar partes do processo
4. O **Cowork** gerencia o projeto inteiro — acompanha dependências, reporta status, executa subtarefas

Isso é um loop fechado. Da ideia ao código em produção, tudo dentro do ecossistema Anthropic.

### O que isso significa na prática

Imagine que você quer criar um micro SaaS. Antes, você precisava de:
- Uma ferramenta de design (Figma)
- Um editor de código (VSCode)
- Um sistema de gerenciamento de projeto (Jira, Notion, Trello)
- Reuniões para alinhar o que cada ferramenta produziu

Agora, em teoria, você faz isso dentro do Claude. Um criador solo. Sem time. Em dias.

Isso não é hype. Isso é o que a Anthropic está construindo.

### O sinal do mercado

A Anthropic saiu de $9 bilhões de ARR em dezembro 2025 para $30 bilhões em abril 2026.

Três vezes em quatro meses.

Isso não acontece só com um bom modelo de linguagem. Isso acontece quando você cria um ecossistema que as pessoas não conseguem largar.

## Bloco 3 — Como Começar Hoje: Casos de Uso Concretos (9min–12min)

**[Visual: tela do Cowork com exemplos reais sendo configurados]**

### Caso 1: criador de conteúdo

Se você faz vídeos sobre tecnologia, você pode configurar o Cowork para:
- Monitorar lançamentos de produtos de IA e te mandar um briefing diário
- Pesquisar dados para um roteiro que você descreveu
- Comparar desempenho de vídeos similares no YouTube e sugerir ângulos

Você vai dormir, e de manhã tem material para 3 vídeos esperando.

### Caso 2: desenvolvedor solo

Se você está construindo um micro SaaS:
- Delega a análise de tickets de suporte
- Configura o Cowork para monitorar issues no GitHub e classificar por prioridade
- Pede para ele comparar seu produto com os 5 concorrentes diretos toda semana

### Caso 3: consultor ou analista

- Coloca um conjunto de dados e pede uma análise semanal automática
- Configura alertas para quando determinadas condições forem detectadas
- Usa o thread persistente como "memória institucional" do projeto

### Como acessar

1. Abra o Claude Desktop (macOS ou Windows — o Web app ainda tem limitações)
2. O Cowork aparece como opção no menu lateral
3. Defina um objetivo, dê o contexto necessário, configure o nível de autonomia
4. Pronto — ele começa a trabalhar
