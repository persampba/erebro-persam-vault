## Bloco 1 — O que é Micro SaaS e por que 2026 é o momento (2min–5min)

### Definição prática
- **SaaS:** Software as a Service — você paga mensalmente por um software na nuvem
- **Micro SaaS:** versão enxuta, de nicho, sem VC, sem equipe grande
- Exemplos: ferramenta de agendamento para salões de beleza, automação de nota fiscal para MEIs, monitor de preço de concorrentes para e-commerce

### Por que 2026 é diferente
- **IA gerou alavancagem de código:** com Claude Code ou Cursor, uma pessoa sem formação técnica pode construir um produto funcional em dias
- **No-code democratizou o resto:** Bubble, Glide, Softr — qualquer pessoa monta interface
- **Agentes de IA são o serviço:** em vez de construir software que executa regras fixas, você constrói agentes que tomam decisões — o produto fica mais inteligente sem você programar cada caso

### O mercado brasileiro
- Menos de 5% das PMEs brasileiras usam SaaS — isso significa que 95% do mercado não tem solução digital para seus problemas
- Comparado com EUA, onde a penetração é >60%, isso é uma janela de 10–15 anos de crescimento pela frente
- O criador de Micro SaaS brasileiro que entrar agora vai colher por anos

## Bloco 2 — As 3 Bolhas do Micro SaaS no Brasil (5min–9min)

O mercado brasileiro de Micro SaaS não é uniforme. Tem 3 grupos bem distintos entrando agora:

### Bolha 1 — Os Devs Independentes
- Perfil: desenvolvedor que cansou de CLT ou freelancer e quer renda recorrente
- Vantagem: habilidade técnica, consegue construir rápido
- Desvantagem: às vezes constroem o que é possível, não o que o mercado quer
- Oportunidade hoje: usar agentes de IA para multiplicar o que conseguem entregar sozinhos

### Bolha 2 — Os Infoprodutores Migrando
- Perfil: criador de conteúdo ou vendedor de cursos que percebeu que infoproduto está saturado
- Vantagem: já tem audiência, já sabe vender
- Desvantagem: zero habilidade técnica — mas isso está mudando com IA
- Oportunidade hoje: usar no-code + agentes de IA para construir ferramentas para a própria audiência

### Bolha 3 — Os Profissionais de Nicho
- Perfil: contador, advogado, dentista, veterinário que entende profundamente de um setor
- Vantagem: conhecimento profundo de um problema real e uma rede de clientes prontos
- Desvantagem: nunca pensou em criar software
- Oportunidade hoje: com Claude Design + Claude Code, esse profissional pode prototipar e construir uma solução para o seu setor sem depender de dev

**Ponto de tensão:** Qual bolha você é? (engaja comentários)

## Bloco 3 — Como Construir um Micro SaaS com Agentes de IA (9min–13min)

### Passo 1 — Encontrar o problema certo
- Regra de ouro: o problema tem que ser doloroso, recorrente e ter disposição de pagar
- Método prático: liste as 3 tarefas mais repetitivas que você faz ou vê alguém fazer na sua área
- Pergunte: "Quanto eu pagaria por mês para nunca mais fazer isso?"
- Se a resposta for acima de R$ 50/mês, tem mercado

### Passo 2 — Validar antes de construir
- Crie uma landing page em 1 dia (use o próprio Claude para escrever o copy)
- Liste o produto no Product Hunt, em grupos do WhatsApp do seu nicho, no LinkedIn
- Meta: 10 pessoas dispostas a pagar antes de escrever uma linha de código
- Se não conseguir 10 pessoas, o problema não é urgente o suficiente

### Passo 3 — Construir com agentes de IA
- **Claude Design** → prototipe a interface em texto, receba mockup visual
- **Claude Code** → descreva o que cada função deve fazer, deixe o modelo escrever o código
- **Agentes de IA no produto:** em vez de programar regras fixas, use a API da Anthropic para que o produto tome decisões — ex: classificar automaticamente mensagens de clientes, gerar relatórios personalizados, sugerir ações baseadas em dados
- **Stack recomendado para não-dev:** Bubble ou Glide para frontend + API Anthropic para inteligência + Stripe para pagamento

### Passo 4 — Precificar e lançar
- Preço inicial sugerido: R$ 97–R$ 197/mês
- 50 clientes = R$ 4.850–R$ 9.850 de MRR
- 100 clientes = R$ 9.700–R$ 19.700 de MRR
- Foco total nos primeiros 50 clientes: atendimento próximo, feedback constante, iteração rápida

### O diferencial dos agentes
- Um Micro SaaS normal executa regras que você programou
- Um Micro SaaS com agentes de IA aprende com o uso e adapta o comportamento
- Isso significa que o produto fica melhor sem você intervir — e isso justifica preço premium
