## Introdução (30s–2min)

### O que é micro SaaS

- **SaaS** = Software as a Service: você paga uma assinatura mensal para usar um software na nuvem (ex: Spotify, Notion, Canva).
- **Micro SaaS** = versão enxuta, focada em resolver **um problema específico** para um **nicho pequeno**.
- Não é escalar para 1 milhão de usuários — é faturar R$ 5 mil, R$ 10 mil, R$ 30 mil por mês com uma solução simples.

### Por que agora é o melhor momento

- No Brasil, apenas **5% das PMEs usam SaaS** — espaço gigante para crescer.
- Ferramentas no-code e IA **derrubaram a barreira técnica**.
- Você não compete com a Microsoft — você resolve um problema que a Microsoft ignora.

## Bloco 1 — A lógica do micro SaaS (2min–5min)

### Os 3 pilares

1. **Problema específico e real** — não invente, encontre uma dor que alguém já tem.
2. **Nicho bem definido** — dentistas, advogados trabalhistas, personal trainers, donos de pet shop.
3. **Receita recorrente** — mensalidade previsível, não venda avulsa.

### Exemplos reais de micro SaaS simples

- Agendamento automático para clínicas de estética com WhatsApp integrado.
- Gerador de contratos para freelancers com IA.
- Dashboard de métricas de Instagram para pequenas agências.
- Sistema de feedback automático para restaurantes via QR Code.

### O modelo financeiro

- 100 clientes pagando R$ 97/mês = **R$ 9.700/mês** de receita recorrente.
- Custo marginal praticamente zero após o produto pronto.
- Churn (cancelamentos) é o inimigo — resolver um problema real reduz churn.

## Bloco 2 — O papel da IA na construção (5min–10min)

### Como a IA muda o jogo para quem não programa

- **Claude e ChatGPT** conseguem escrever código funcional a partir de uma descrição em português.
- Demonstração ao vivo: pedir para o Claude criar a lógica de um formulário de agendamento.
- Ferramentas no-code que integram IA: **Bubble**, **Glide**, **Softr**, **Buildship**.

### O fluxo de trabalho prático

1. **Ideação com IA:** "Claude, me dê 10 ideias de micro SaaS para o mercado de clínicas veterinárias brasileiras."
2. **Validação:** postar em grupos, perguntar para potenciais clientes, criar landing page simples.
3. **Construção:** usar Bubble ou Glide + Claude para escrever lógica e automações.
4. **Cobrança:** Stripe, Hotmart, Kiwify — integração simples.
5. **Lançamento:** primeiros 10 clientes como beta com desconto, coleta de feedback.

### Demonstração de tela

- Mostrar o Claude gerando um código de API simples.
- Mostrar uma tela do Bubble com automação montada.
- Mostrar uma landing page feita no Carrd ou Framer em 30 minutos.

### Quanto tempo leva?

- Validar ideia: **1 semana**.
- Construir MVP: **2 a 4 semanas** com ferramentas no-code + IA.
- Primeiros clientes pagantes: **30 a 60 dias**.

## Bloco 3 — Os erros que matam micro SaaS antes de começar (10min–13min)

### Erro 1: construir sem validar

- A maioria das pessoas passa meses construindo algo que ninguém quer comprar.
- Regra: **fale com 10 potenciais clientes antes de escrever uma linha de código.**
- Validação mínima: landing page + formulário de interesse + 3 pré-vendas.

### Erro 2: escolher nicho amplo demais

- "Software para empresas" não é nicho. "Software de controle de estoque para açougues no interior" é nicho.
- Quanto mais específico, menos concorrência e mais fácil de vender.

### Erro 3: preço baixo demais

- Cobrar R$ 29/mês parece mais fácil de vender, mas você precisa de 345 clientes para fazer R$ 10k.
- Cobrando R$ 197/mês, você precisa de apenas 50 clientes.
- **Preço reflete percepção de valor** — caro e que resolve bem vende mais que barato e genérico.

### Erro 4: ignorar o suporte

- Micro SaaS vive de reputação e boca a boca.
- Tempo de resposta ao cliente é o diferencial de quem não tem time de vendas.
